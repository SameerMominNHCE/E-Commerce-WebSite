## 🎯 PROJECT OVERVIEW

**LocalStore E-commerce Platform** is a full-stack web application that enables customers to browse, search, and purchase products online from a local store. The platform includes:

- **Customer Features**: Browse products, add to cart, checkout, track orders, leave reviews
- **Admin Features**: Manage products (CRUD), track inventory, manage orders
- **Authentication**: Secure login/registration system
- **Responsive Design**: Works on desktop, tablet, and mobile devices
- **Real-time Updates**: Live order tracking and inventory management

## 📦 INSTALLATION GUIDE

### Step 1: Install Node.js and npm

**Windows:**
1. Go to https://nodejs.org/
2. Download LTS version
3. Run the installer
4. Follow the installation wizard
5. Verify installation:
```bash
node --version
npm --version
```

**macOS:**
```bash
# Using Homebrew
brew install node
node --version
npm --version
```

**Ubuntu/Linux:**
```bash
# Using apt-get
sudo apt update
sudo apt install nodejs npm
node --version
npm --version
```

### Step 2: Install MongoDB

**Windows:**
1. Go to https://www.mongodb.com/try/download/community
2. Download Windows MSI installer
3. Run the installer
4. Choose "Install MongoDB as a Service"
5. Complete installation

**macOS:**
```bash
# Using Homebrew
brew tap mongodb/brew
brew install mongodb-community
brew services start mongodb-community
```

**Ubuntu/Linux:**
```bash
# Import MongoDB GPG key
wget -qO - https://www.mongodb.org/static/pgp/server-5.0.asc | sudo apt-key add -

# Add MongoDB repository
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/5.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-5.0.list

# Install MongoDB
sudo apt-get update
sudo apt-get install -y mongodb-org

# Start MongoDB service
sudo systemctl start mongod
sudo systemctl enable mongod
```

### Step 3: Clone the Project

```bash
# Clone repository (if using Git)
git clone <repository-url>
cd ecommerce-platform

# Or create project manually
mkdir ecommerce-platform
cd ecommerce-platform
```

### Step 4: Setup Backend

```bash
# Navigate to server directory
cd server

# Install dependencies
npm install

# Verify installation
npm list
```

### Step 5: Setup Frontend

```bash
# Navigate to client directory (from root)
cd client

# Install dependencies
npm install

# Verify installation
npm list
```

## 🔧 CONFIGURATION SETUP

### Backend Configuration

**Create `server/.env` file:**

```env
# Server Configuration
PORT=5000
NODE_ENV=development

# MongoDB Configuration
MONGODB_URI=mongodb://localhost:27017/ecommerce

# JWT Configuration
JWT_SECRET=your_super_secret_jwt_key_change_this_in_production
JWT_EXPIRE=7d

# Email Configuration (Optional)
EMAIL_USER=your_email@gmail.com
EMAIL_PASSWORD=your_app_password

# CORS Configuration
CORS_ORIGIN=http://localhost:3000

# Stripe Configuration (Optional for payments)
STRIPE_PUBLIC_KEY=your_stripe_public_key
STRIPE_SECRET_KEY=your_stripe_secret_key
```

**Generate a secure JWT Secret:**
```bash
# Run in Node.js terminal
require('crypto').randomBytes(32).toString('hex')
```

### Frontend Configuration

**Create `client/.env` file:**

```env
# API Configuration
VITE_API_URL=http://localhost:5000

# Application Configuration
VITE_APP_NAME=LocalStore
VITE_APP_VERSION=1.0.0
```

**Create `client/vite.config.js`:**

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:5000',
        changeOrigin: true
      }
    }
  }
})
```


```
➜  Local:   http://localhost:3000/
➜  Network: use --host to expose
➜  press h + enter to show help
```

### Method 2: Using Concurrently (Run Both from Root)

**Install concurrently:**
```bash
npm install -g concurrently
```

**Create `package.json` in root directory:**
```json
{
  "name": "ecommerce-platform",
  "scripts": {
    "dev": "concurrently \"cd server && npm run dev\" \"cd client && npm run dev\"",
    "server": "cd server && npm run dev",
    "client": "cd client && npm run dev",
    "build": "cd client && npm run build",
    "setup": "cd server && npm run create-admin && npm run create-products"
  }
}
```

**Run everything at once:**
```bash
npm run dev
```

## 📡 API ENDPOINTS

### Authentication Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/register` | Register new user |
| POST | `/api/auth/login` | Login user |
| GET | `/api/auth/me` | Get current user |
| PUT | `/api/auth/profile` | Update profile |

**Example:**
```bash
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "password123"
  }'
```

### Product Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/products` | Get all products |
| GET | `/api/products/:id` | Get single product |
| POST | `/api/products` | Create product (admin) |
| PUT | `/api/products/:id` | Update product (admin) |
| DELETE | `/api/products/:id` | Delete product (admin) |
| GET | `/api/products/categories/list` | Get all categories |

**Example:**
```bash
# Get all products
curl http://localhost:5000/api/products?page=1&limit=12

# Get with filters
curl "http://localhost:5000/api/products?category=Electronics&minPrice=1000&maxPrice=5000&sortBy=price_low"
```

### Cart Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/cart` | Get cart (auth required) |
| POST | `/api/cart/add` | Add to cart (auth required) |
| PUT | `/api/cart/update/:itemId` | Update quantity (auth required) |
| DELETE | `/api/cart/remove/:itemId` | Remove from cart (auth required) |
| DELETE | `/api/cart/clear` | Clear cart (auth required) |

### Order Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/orders/create` | Create order (auth required) |
| GET | `/api/orders` | Get user orders (auth required) |
| GET | `/api/orders/:orderId` | Get order details (auth required) |
| PUT | `/api/orders/:orderId/cancel` | Cancel order (auth required) |
| PUT | `/api/orders/:orderId/status` | Update status (admin only) |

### Review Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/reviews/product/:productId` | Get product reviews |
| POST | `/api/reviews/add` | Add review (auth required) |
| PUT | `/api/reviews/:reviewId/helpful` | Mark as helpful (auth required) |

## 🐛 TROUBLESHOOTING

### Common Issues and Solutions

#### 1. **MongoDB Connection Error**

**Error:**
```
MongooseError: Cannot connect to MongoDB
```

**Solutions:**
- Make sure MongoDB service is running
- Check MongoDB URI in `.env`
- Verify port 27017 is accessible
- Check firewall settings

**Windows:**
```bash
# Check if MongoDB service is running
net start MongoDB

# Or in Services.msc - look for "MongoDB Server"
```

**macOS:**
```bash
# Start MongoDB service
brew services start mongodb-community

# Check status
brew services list
```

**Linux:**
```bash
# Start MongoDB service
sudo systemctl start mongod

# Check status
sudo systemctl status mongod
```

#### 2. **Port Already in Use**

**Error:**
```
Error: listen EADDRINUSE :::5000
```

**Solutions:**
```bash
# Kill process on port 5000 (Windows)
netstat -ano | findstr :5000
taskkill /PID <PID> /F

# Kill process on port 5000 (macOS/Linux)
lsof -i :5000
kill -9 <PID>

# Or change port in server/.env
PORT=5001
```

#### 3. **npm Install Fails**

**Error:**
```
npm ERR! code ERESOLVE
npm ERR! ERESOLVE unable to resolve dependency tree
```

**Solutions:**
```bash
# Clear npm cache
npm cache clean --force

# Install with legacy peer deps
npm install --legacy-peer-deps

# Or use npm ci
npm ci
```

#### 4. **CORS Error**

**Error:**
```
Access to XMLHttpRequest blocked by CORS policy
```

**Solutions:**
- Make sure backend and frontend URLs match in `.env`
- Check CORS configuration in `server.js`
- Verify proxy settings in `vite.config.js`

```javascript
// server.js
app.use(cors({
  origin: 'http://localhost:3000',
  credentials: true
}))
```

#### 5. **JWT Token Invalid**

**Error:**
```
Token is not valid
```

**Solutions:**
- Clear browser cookies/localStorage
- Re-login
- Check JWT_SECRET is same in backend `.env`
- Verify token is being sent in Authorization header

#### 6. **React Component Not Rendering**

**Error:**
```
[vite] error during import: Failed to fetch dynamically imported module
```

**Solutions:**
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

# Clear Vite cache
rm -rf .vite

# Restart dev server
npm run dev
```

#### 7. **Product Images Not Loading**

**Error:**
```
Failed to load image from placeholder.com
```

**Solutions:**
- Use valid image URLs
- Create products with real image URLs
- Or upload to image hosting service:
  - imgur.com
  - cloudinary.com
  - tinypng.com

#### 8. **Authentication Issues**

**Error:**
```
Invalid credentials
```

**Solutions:**
- Verify email and password are correct
- Check database has user created
- Clear browser storage and retry

```bash
# Recreate admin user
npm run create-admin
```

```bash
npm install -g vercel

# Deploy
vercel

# Set environment variables in Vercel dashboard
# VITE_API_URL=https://your-backend.herokuapp.com
```

### Deploy Database (MongoDB Atlas)

1. Create free cluster on mongodb.com/cloud/atlas
2. Get connection string
3. Update MONGODB_URI in backend `.env`
